Why it’s useful

A tiny UX upgrade with surprisingly satisfying results.

  • Map any CSS selector to a haptic pattern
  • Use one plugin class across blocks, templates, and custom HTML
  • Pick from built-in presets or write custom millisecond patterns
  • Preview behavior on desktop with visual + audio fallback

Uses the same client-side haptic core as the plugin.

Functionality overview

Built for real UI interactions, not gimmicky buzz-buzz chaos

The plugin focuses on practical feedback patterns: confirmation taps, warning pulses, button emphasis, and richer micro-interactions for mobile-first experiences.

How setup usually works

Short version: pick targets, choose feel, save, done.

  1. Add a rule and enter a selector like .buy-now or #checkout.
  2. Choose a preset or define a custom timing sequence in milliseconds.
  3. Select the trigger such as click/tap, mouse down, or touch start.
  4. Optionally enable the plugin class for reusable markup patterns.
  5. Use Desktop Debug Mode while testing on browsers without mobile haptics.
